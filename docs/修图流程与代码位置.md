# 修图流程与代码位置

本文档说明 EasyStitch 中 **修图** 模块的完整流程及对应代码位置，便于后续功能添加与维护。

---

## 一、流程总览

```
首页(HomeView) → 点击「进入修图」→ EditCanvas
                    ↓
        无项目：空状态 +「导入/新建项目」
        有项目：DiamondCanvas（画布 + 色卡管理）
                    ↓
        项目管理弹窗(ProjectManager)
          ├─ 新建项目(ProjectForm)：项目信息、图片导入、网格设置&颜色配置
          └─ 历史项目(XmlSnapshotList)：导入 XML 快照
                    ↓
        创建/导入成功 → currentProject 写入 → 进入画布编辑
                    ↓
        DiamondCanvas
          ├─ CanvasHeader：显示模式、透明、边框、撤销/重做、保存、缩放、吸管、填充等
          ├─ 左侧 CanvasToolbar：项目信息、色卡选择(ColorPalette)
          ├─ 中间：canvas 画布（像素格编辑）
          └─ 右侧 ColorStatistics：颜色统计、Code/No./Bag 表
```

---

## 二、入口与路由（无 Vue Router，用 mode 切换）

| 步骤 | 说明 | 代码位置 |
|------|------|----------|
| 首页 | 两个卡片：「进入修图」「进入合图」 | `src/components/HomeView.vue` |
| 模式切换 | 点击「进入修图」触发 `go-edit`，App 设置 `mode='edit'` | `src/App.vue`（`enter('edit')` → `viewComponent = EditCanvas`） |
| 修图主容器 | 顶部「项目」按钮 +「色卡管理」按钮；中间画布或空状态 | `src/components/EditCanvas.vue` |

**关键文件：**
- `src/App.vue`：`mode` 状态，`viewComponent` 计算属性，`EditCanvas` 组件
- `src/components/HomeView.vue`：`@click="$emit('go-edit')"` 的修图卡片
- `src/components/EditCanvas.vue`：修图页唯一根组件，挂载 `ProjectManager`、`ColorPanel`、`DiamondCanvas`

## 三、项目管理（新建项目 / 历史项目）

### 3.1 打开方式

- 在 EditCanvas 顶部点击 **「项目」**，或无项目时点击 **「导入/新建项目」** → `showProjectManager = true`，弹出项目管理弹窗。

### 3.2 项目管理弹窗

| 功能 | 说明 | 代码位置 |
|------|------|----------|
| 弹窗本体 | 标题「项目管理」，两个 Tab | `src/components/Project/ProjectManager.vue` |
| 新建项目 Tab | 内嵌 ProjectForm，底部「创建项目」调用 form 的 submit | `ProjectManager.vue` 中 `<ProjectForm ref="formRef" @submit="handleCreate" />` |
| 历史项目 Tab | 导入 XML 快照列表 | `ProjectManager.vue` 中 `<XmlSnapshotList @import-xml="onImportXml" />` |
| 创建逻辑 | `handleCreate(data)` → `useLocalProjects().create(data)` → `emit('project-created', res.data)` | `ProjectManager.vue` + `src/composables/useLocalProjects.js` |
| 导入 XML | `onImportXml(payload)` 解析 cellsMatrix、原图等，构造临时项目并 `emit('project-imported', tempProject)` | `ProjectManager.vue` 内 |

### 3.3 新建项目表单（项目信息、图片导入、网格与颜色）

| 区块 | 说明 | 代码位置 |
|------|------|----------|
| 项目信息 | 项目名称输入 | `src/components/Project/ProjectForm.vue`（第一个 el-card） |
| 图片导入 | 点击/拖拽上传，预览、重新选择、删除，显示尺寸与比例 | `ProjectForm.vue`（第二个 el-card，`upload-section`） |
| 网格设置 | 网格长度、网格宽度（可自动计算） | `ProjectForm.vue`（第三个 el-card 内「网格设置」） |
| 颜色配置 | 颜色组、颜色数量等 | `ProjectForm.vue`（同一 card 内「颜色配置」） |

**项目持久化：**
- `src/composables/useLocalProjects.js`：`create / loadPage / search / remove / select`，内部调用 `projectStore.js`
- `src/database/indexeddb/projectStore.js`：IndexedDB 读写
- `src/utils/projectDataConverter.js`：前端格式 ↔ 后端/存储格式（如 `convertToBackendFormat`、`convertToFrontendFormat`）

## 四、修图画布（有项目时的主界面）

当 `currentProject` 有值后，EditCanvas 渲染 **DiamondCanvas**，并传入 `projectData="currentProject"`。

### 4.1 画布整体结构

| 区域 | 说明 | 代码位置 |
|------|------|----------|
| 画布容器 | 三列：左工具栏 / 中间画布 / 右侧统计 | `src/components/DiamondCanvas/DiamondCanvas.vue` |
| 顶部栏 | 显示模式、透明、边框、撤销/重做、保存、缩放、吸管、填充、策略等 | `src/components/DiamondCanvas/CanvasHeader.vue` |
| 左侧栏 | 项目信息（tooltip）、色卡选择 | `src/components/DiamondCanvas/CanvasToolbar.vue` |
| 中间 | canvas 渲染、吸管浮层、滚动占位 | `DiamondCanvas.vue` 内 `canvasRef`、`canvasStageRef` |
| 右侧栏 | 颜色统计（已用颜色、格子数、Code/No./Bag 表） | `src/components/DiamondCanvas/ColorStatistics.vue` |

### 4.2 画布核心逻辑（像素格、历史、导出）

- **画布引擎**：`src/core/diamond_canvas.js`（DiamondCanvas 类，格子数据、绘制、点击编辑、导出等）
- **撤销/重做**：`src/components/DiamondCanvas/useCanvasHistory.js`，与 `CanvasHeader` 的撤销/重做按钮联动，并会 `refreshColorStatistics`
- **颜色统计刷新**：`DiamondCanvas.vue` 中 `refreshColorStatistics()` 调用 `ColorStatistics.refreshData()`
- **保存/导出**：`CanvasHeader` 的保存 → `handleSaveProject`，会用到 `src/utils/projectXml.js`（如 `buildProjectXml`、`downloadXml`）等

### 4.3 色卡与颜色管理

| 功能 | 说明 | 代码位置 |
|------|------|----------|
| 顶部「色卡管理」按钮 | 在 EditCanvas 头部，点击打开色卡管理中心 | `src/components/ColorPanel.vue`（当前主要为「色卡管理」按钮） |
| 色卡管理中心弹窗 | 色卡组/色卡的增删改查 | `src/components/ColorManagementCenter.vue` |
| 画布左侧色卡选择 | 在画布内选当前使用的颜色 | `src/components/DiamondCanvas/ColorPalette.vue`，由 CanvasToolbar 引入 |
| 颜色数据与调色板 | 全局色卡加载、当前调色板、选色索引 | `src/composables/useColorManagement.js` |
| 颜色分组 | 按组加载色卡（如「全部颜色」） | `src/composables/useColorGroups.js`、`useColorGroupsForPanel.js` |
| 色卡/颜色 API | 与后端或本地数据交互 | `src/api/colorAPI.js`、`colorsAPI.js`、`colorGroupAPI.js` |

## 五、数据流与状态（修图相关）

### 5.1 项目数据

- **当前项目**：`EditCanvas.vue` 中 `currentProject`（ref）。创建/导入后通过 `handleProjectCreated` / `handleProjectImported` 赋值。
- **项目数据结构**（前端）：  
  `name`、`image`（含 `data`/`thumbnail`、`size`）、`grid`（`length`、`width`、`cellSize`）、`colorConfig`（`type`、`colorGroupId`、`colorCount`、`selectedColors`）、`cellsMatrix`、`rows`、`cols` 等，详见 `projectDataConverter.js` 与 `useLocalProjects` 的 create 入参。

### 5.2 画布内状态与 Composables

- **EditCanvas 使用的 composables**：  
  `useColorManagement`、`useCanvasSettings`、`useImageProcessing`、`useLayoutManagement`（见 `EditCanvas.vue` setup）。
- **DiamondCanvas 内**：  
  `useColorManagement`、`useColorGroups`、`useCanvasHeader`、`useCanvasHistory`、`useCanvasResize`、`useEyedropper` 等；画布实例来自 `src/core/diamond_canvas.js`。
- **生成钻石图（若从图片生成格子）**：  
  `useImageProcessing.generateDiamondArt` 在 EditCanvas 中调用，内部会用到 `useCanvasSettings` 和 `colorPalette`。

### 5.3 颜色与策略

- 取色策略（如 Floyd-Steinberg、平均等）：`src/core/color/strategies/`（如 `FloydSteinbergStrategy.js`、`AverageStrategy.js`）、`factory.js`。
- 调色板构建/色域：`src/core/color/palette_builder.js`、`color_space.js`、`kmeans.js`。

## 七、小结

- **修图入口**：`HomeView` → `App`（mode=edit）→ `EditCanvas`。
- **无项目时**：显示「欢迎来到修图」和「导入/新建项目」，点击打开 `ProjectManager`。
- **项目管理**：`ProjectManager` = 新建（`ProjectForm`）+ 历史（`XmlSnapshotList`）；创建走 `useLocalProjects().create()`，导入 XML 在 `ProjectManager.onImportXml` 内构造项目并 `project-imported`。
- **有项目时**：`EditCanvas` 渲染 `DiamondCanvas`，传入 `currentProject`。画布左侧选色、中间编辑、右侧颜色统计；顶部「色卡管理」打开 `ColorManagementCenter`。
- **扩展新功能时**：按上表找到对应模块（如改新建表单→ProjectForm，改画布行为→diamond_canvas.js 或 DiamondCanvas.vue，改颜色统计→ColorStatistics.vue），再结合 `useLocalProjects`、`useColorManagement` 等 composables 即可。
